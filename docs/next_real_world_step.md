# Next Real-World Step

> **The core next step — execute and validate the project in Snowflake — has been completed
> (2026-07-02: 18/18 setup verification + 21/21 reconciliation/DQ; see
> [`validation_results.md`](validation_results.md)).**

The project is built, executed, and validated. The steps below remain a **reusable runbook** for
re-running it in any Snowflake account; the remaining forward step is the optional Cursor rebuild
(Step 6).

## Priority order (do these in sequence)

```
1. Run the current repo live in Snowflake
2. Debug any execution issues
3. Run validation scripts
4. Capture screenshots / evidence
5. Update README + validation documents
6. Rebuild selected parts in Cursor later (deeper learning) — NOT before step 5
```

> **Snowflake execution comes before any Cursor rebuild.** The Cursor rebuild is a separate
> learning exercise for later. Do not start it until the Snowflake execution and validation
> above are complete, or until you explicitly decide to switch.

---

## Step 1 — Set up Snowflake
- Create or open a Snowflake trial/account ([signup](https://signup.snowflake.com/)).
- Open **Snowsight** → **Worksheets** → new worksheet.
- Confirm context: `SELECT CURRENT_ROLE(), CURRENT_WAREHOUSE(), CURRENT_DATABASE();` (use
  `ACCOUNTADMIN` for setup).

## Step 2 — Execute the project scripts (in order)
```
snowflake/00_setup/          -- warehouses, DB, schemas, roles
snowflake/01_ingestion/      -- formats, stage, RAW tables, 05_generate_synthetic_data.sql
snowflake/02_staging/        -- typed/cleaned staging
snowflake/03_core_model/     -- dimensions + facts
snowflake/04_aml_rules/      -- alert seed, generation, scoring
snowflake/05_str_workflow/   -- STR cases + SLA
snowflake/06_reporting/      -- reporting views
snowflake/07_data_quality/   -- validation
```
Optional, later:
```
snowflake/08_automation/     -- Streams & Tasks (leave task suspended)
snowflake/09_snowpark/       -- Snowpark Python example
snowflake/10_powerbi/        -- connect Power BI to REPORTING
```
> Governance (`00_setup/04_governance_policies.sql`) runs **after** the core model exists.
> Full detail: [`manual_snowflake_test_plan.md`](manual_snowflake_test_plan.md) and
> [`deployment_guide.md`](deployment_guide.md).
